# ADR-010: Shadow Mode Implementation

## Status
Proposed (Generated during Phase 0 Architecture Audit)

## Context
ADR-005 (Shadow Traffic Design) was approved and describes the following behavior when `DATABASE_PROVIDER=SHADOW`:
1. Every query is executed on Firestore synchronously
2. The user response is returned immediately
3. An asynchronous worker execution block is scheduled (via `setImmediate` or fire-and-forget Promises) to run the equivalent query against PostgreSQL
4. Latencies, query outcomes, and errors of the shadow query are logged under `[SHADOW_TRAFFIC_LOG]`

However, the actual implementation in all 5 unified repositories treats SHADOW identically to FIRESTORE:
```javascript
if (dbConfig.isFirestore() || dbConfig.isShadow()) {
  return FirestoreXxxRepository.findByXxx(...); // No PG execution at all
}
```

This means **zero shadow metrics are being collected** and PostgreSQL performance under production-like load is completely unvalidated. The Production Migration Governance addendum requires comprehensive Shadow Metrics (Success Rate, Failure Rate, Latency Difference, Data Difference, Schema Difference).

## Decision

**Implement full Shadow Mode per ADR-005 with structured metrics capture.**

### Implementation Design

```javascript
// Unified repository pattern for Shadow Mode
if (flags.ENABLE_SHADOW_MODE) {
  const startTime = Date.now();
  const fsResult = await firestoreRepo.operation(params);

  // Fire-and-forget: async PG shadow execution
  setImmediate(async () => {
    const shadowStart = Date.now();
    try {
      const pgResult = await postgresRepo.operation(params);
      const shadowLatency = Date.now() - shadowStart;

      // Compare results
      const diff = compareResults(fsResult, pgResult);

      logger.info('shadow_traffic', {
        operation: 'XxxRepository.findByXxx',
        shadow_latency_ms: shadowLatency,
        primary_latency_ms: Date.now() - startTime,
        success: true,
        data_match: Object.keys(diff).length === 0,
        diff_fields: diff,
      });
    } catch (err) {
      logger.warn('shadow_traffic', {
        operation: 'XxxRepository.findByXxx',
        shadow_latency_ms: Date.now() - shadowStart,
        success: false,
        error: err.message,
        error_code: err.code,
      });
    }
  });

  return fsResult; // User gets Firestore result immediately
}
```

### Shadow Metrics Collected

| Metric | Description | Log Key |
|---|---|---|
| Success Rate | % of shadow operations that complete | `success: true/false` |
| Failure Rate | % of shadow operations that throw | `success: false` |
| Latency Difference | FS latency vs PG shadow latency | `primary_latency_ms` vs `shadow_latency_ms` |
| Data Difference | Fields that differ between results | `diff_fields: {field: {fs, pg}}` |
| Schema Difference | Missing/extra fields | `diff_fields` |
| Error Code | PostgreSQL error code on failure | `error_code` |

### Connection Pool Safety
- Shadow queries use a SEPARATE PostgreSQL connection pool
- Configured with lower max connections (25% of primary pool)
- Statement timeout: 5 seconds (shadow queries killed if too slow)
- Event loop lag monitored; shadow execution skipped if lag > 50ms

## Alternatives Considered

**Option A: Synchronous Shadowing.**
- Run PG query synchronously before returning FS result
- **Rejected:** Directly increases user-facing latency by PG execution time

**Option B: Log Replay (offline).**
- Record requests, replay on separate server
- **Rejected:** Does not test PG under real concurrent production load and connection pool conditions

**Option C: Skip Shadow Mode entirely.**
- Rely on DUAL_READ/DUAL_WRITE for validation
- **Rejected:** DUAL modes impact user latency; Shadow Mode provides safe async validation

## Consequences
- **Safety:** User latency unaffected; PG connection pool protected
- **Validation:** Real production query patterns tested on PostgreSQL
- **Telemetry:** All 6 required shadow metrics captured
- **Overhead:** Additional PostgreSQL pool (low cost) + setImmediate scheduling
- **Log Volume:** Shadow logs may be high; use sampling in high-traffic scenarios

## Risks
- **Event Loop Saturation:** Too many async shadow operations could block the event loop
  - Mitigation: Skip shadow if event loop lag > 50ms; monitor with `perf_hooks`
- **Memory Pressure:** Shadow result objects held in closures
  - Mitigation: Compare and release; GC collects after setImmediate callback
- **Connection Pool Exhaustion:** Separate pool prevents shadow from starving primary



## Runtime Provider Registry (Required by Architecture Review)

A lightweight registry pattern enables test injection and runtime provider switching:

```javascript
// config/providerRegistry.js
class ProviderRegistry {
  constructor() {
    this._providers = new Map();
  }

  register(name, adapter) {
    this._providers.set(name, adapter);
  }

  get(name) {
    const adapter = this._providers.get(name);
    if (!adapter) throw new Error(`Provider '${name}' not registered`);
    return adapter;
  }

  has(name) {
    return this._providers.has(name);
  }
}

// Singleton instance
const registry = new ProviderRegistry();

// Register adapters at startup
registry.register('firestore', require('../repositories/firestore/PhoneRepository'));
registry.register('postgres', require('../repositories/postgres/PhoneRepository'));

module.exports = registry;
```

**Usage in tests:**
```javascript
const registry = require('../config/providerRegistry');
registry.register('firestore', mockFirestoreAdapter);
registry.register('postgres', mockPostgresAdapter);
```

## Admin Endpoint Specification (Required by Architecture Review)

```
POST /api/v1/admin/feature-flags
```

**Authentication:** `requireRole(['admin'])` middleware (from `middleware/auth.js`)

**Request Body:**
```json
{
  "ENABLE_SHADOW_MODE": true,
  "ENABLE_DUAL_WRITE": false
}
```

**Validation:**
- Only recognized flag keys accepted (reject unknown keys)
- Flag combinations validated per ADR-009 rules
- Invalid combinations return 400 with specific error message

**Atomicity:**
- All flags read from a single `Object.assign()` operation
- No partial updates possible
- Previous state captured before mutation

**Audit:**
```javascript
logger.info('feature_flag_change', {
  actor: req.user.uid,
  previous: { ENABLE_SHADOW_MODE: false },
  current: { ENABLE_SHADOW_MODE: true },
  timestamp: new Date().toISOString(),
});
```

**Response (200):**
```json
{
  "success": true,
  "previous": { "ENABLE_SHADOW_MODE": false },
  "current": { "ENABLE_SHADOW_MODE": true },
  "applied_at": "2026-07-08T12:00:00Z"
}
```

**Response (400 — invalid combination):**
```json
{
  "success": false,
  "error": "ENABLE_SHADOW_MODE requires ENABLE_POSTGRES=true"
}
```

## Test Provider Selection (Required by Architecture Review)

Tests MUST NOT depend on live database connections. Provider selection for tests:

```javascript
// test/setup.js
beforeAll(() => {
  // Force all flags to known state
  process.env.ENABLE_FIRESTORE = 'true';
  process.env.ENABLE_POSTGRES = 'true';
  process.env.ENABLE_DUAL_WRITE = 'false';
  process.env.ENABLE_DUAL_READ = 'false';
  process.env.ENABLE_SHADOW_MODE = 'false';

  // Register mock adapters
  const registry = require('../config/providerRegistry');
  registry.register('firestore', createMockFirestoreAdapter());
  registry.register('postgres', createMockPostgresAdapter());
});

afterAll(() => {
  // Restore real adapters
  registry.register('firestore', require('../repositories/firestore/...'));
  registry.register('postgres', require('../repositories/postgres/...'));
});
```

**Contract Test Pattern:**
```javascript
describe.each([
  ['firestore', firestoreAdapter],
  ['postgres', postgresAdapter],
])('PhoneRepository contract — %s', (name, adapter) => {
  test('findByHash returns null for unknown hash', async () => {
    const result = await adapter.findByHash('nonexistent');
    expect(result).toBeNull();
  });

  test('upsert creates and findByHash retrieves', async () => {
    await adapter.upsert('hash123', { riskScore: 50 });
    const result = await adapter.findByHash('hash123');
    expect(result.riskScore).toBe(50);
  });
});
```

## Shadow Sampling Strategy (Required by Architecture Review)

For high-traffic production scenarios, shadow execution can be sampled:

```javascript
// config/featureFlags.js
SHADOW_SAMPLE_RATE: parseFloat(process.env.SHADOW_SAMPLE_RATE || '1.0'),

// In repository
if (flags.ENABLE_SHADOW_MODE && Math.random() < flags.SHADOW_SAMPLE_RATE) {
  setImmediate(async () => { /* shadow PG execution */ });
}
```

- **Default:** 1.0 (100% — shadow everything)
- **High Traffic:** 0.1 (10% — statistically significant sample)
- **Low Traffic:** 1.0 (100% — every operation matters)
- **Metric:** `shadow_sampled: true/false` in log to distinguish sampled vs unsampled
